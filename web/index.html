how it works

Start a worker.
Give it a job.

Workers are persistent daemons that run on a schedule, respond to messages, and execute bash — on any model.

template.yaml
name: news-worker model: ollama/qwen2.5:7b # free, local schedule: every 1h schedule_script: | # fetch RSS, summarize, notify — no LLM needed for this part python3 fetch_news.py | notify-send "Daily Brief" on_message: Help the user. Use bash for real data.
terminal
$ meshclaw start ./news-worker/ ✓ news-worker is running (pid 3140962) socket: /tmp/meshclaw-news-worker.sock $ meshclaw ask news-worker "오늘 AI 뉴스 요약해줘" 1. GPT-5.4 Pro가 미해결 수학 문제 해결... 2. FCC, 해외산 라우터 규제 강화...
features

Everything you need.
Nothing you don't.

Built for people who want AI agents that actually work, not architecture diagrams.

🖥️

Runs anywhere

MacBook, GPU server, Raspberry Pi, cloud VM. If it runs Python 3.9+, it runs meshclaw.

🆓

Free models

Use Ollama with local LLMs — no API key, no cost, no rate limits. Or bring your own API.

⏰

Scheduled tasks

Run scripts on a schedule — hourly, daily, on demand. LLM optional.

🔀

Orchestration

One boss worker distributes jobs to a fleet. MacBook orchestrates GPU servers.

🔧

Bash access

Workers execute real bash commands. No sandboxed pretend environments.

📡

Unix socket IPC

Workers communicate over Unix sockets. Fast, local, no network overhead.
